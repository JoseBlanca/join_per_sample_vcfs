# ng tandem-repeat scanner — Milestone D (trf-mod parity validation)

*Implementation report, 2026-07-14. Plan:
[`doc/devel/ng/impl_plan/ssr_repeat_scanner.md`](../../ng/impl_plan/ssr_repeat_scanner.md)
(Milestone D, steps D1–D2). Design: [spec](../../ng/spec/ssr_repeat_scanner.md) §6, §9.
**Production stays on `trf-mod`** — this milestone *validates* the scanner; it changes no
shipping code.*

## What landed

- **D1 — golden fixture.** A deterministic synthetic STR-diversity reference
  (`tests/data/tandem_repeat/synthetic_ref.fa`, generated by the committed
  `generate_synthetic_ref.py`): two contigs of aperiodic filler with ~19 embedded tracts —
  perfect and imperfect, periods 2–6, varied copy number, plus a homopolymer and a compound
  motif. The current `trf-mod` → `postprocess` path (`ssr-catalog`) was run on it in the
  container to produce the committed golden catalog `golden.ssr_catalog.bed.gz` (**16 loci**;
  the homopolymer and compound `ATAT` correctly dropped).
- **D2 — parity test.** `src/ssr/catalog/scanner_parity.rs`: runs `find_tandem_repeats` on the
  same reference, bridges the intervals into the **unchanged** `build_loci` via the test-only
  `TrfRecord::for_test`, and asserts the resulting `Locus` set reproduces the golden one.
  Overlap-based recall (spec §6.1: boundary/phase wobble between two detectors is expected), with
  an exact / wobble / scanner-only breakdown.

## Result

**16/16 recall (100 %)** — 15 loci byte-exact, 1 boundary/phase wobble (the GAA tract: scanner
`ctg1:1020-1059 GAA` vs golden `ctg1:1021-1057 AAG` — same tract, ±1–2 bp, rotated motif), and
**1 scanner-only** locus (`ctg2:200-239 CCG`, a genuine STR `trf-mod`'s significance model
rejected — the scanner is *more* sensitive there). Comfortably clears the ≥ 99 % parity bar.

## The key validated finding — the catalog needs a pre-`build_loci` filter

`trf-mod` hands `postprocess` a **clean** candidate set; the raw scanner is deliberately
permissive (`min_copies = 2`), so it also emits low-copy noise (in aperiodic sequence) and every
period-multiple of a real tract. Fed straight to `build_loci`, that noise trips `drop_bundles`
— which runs **before** the copy-number floor — and cascades the *real* loci away (first run:
**0/16**). The fix is **catalog policy**, applied before `build_loci`, mirroring what `trf-mod`
bakes in:

1. the **per-period copy floor** (`{2:5, 3:4, else 3}`) — removes low-copy noise before bundling; and
2. **period-multiple redundancy elimination** (`trf-mod`'s `IsRedundant`) — drops a higher-period
   interval overlapping a divisor-period one (the same tract seen at a multiple period).
   Without it, a period-6 view of the GAA tract whose left boundary had drifted into the flank
   (a coincidental match) survived `postprocess`'s compound-drop and bundle-dropped the real
   period-3 tract.

With both (0/16 → 15/16 → 16/16) the scanner reproduces the catalog. This filter belongs in
`catalog::run` when the production swap lands (spec §6, §7 — deferred); it stays out of the
use-agnostic scanner and the unchanged post-filter. Recorded in the spec §6 and the parity
test's module docs.

## Out of scope (per the revised plan)

The end-to-end `ssr_tomato1` genotyping check needs a scanner-*built* production catalog — i.e.
the deferred `trf-mod` swap — so it is revisited with that decision (D2's locus-set parity
already proves catalog equivalence).

`cargo fmt --check` clean; `cargo clippy --lib --tests -- -D warnings` clean;
`cargo test --lib ssr::catalog::scanner_parity` → **1 pass** (16/16 recall). Golden generated
with the container's `trf-mod` (v4.10.0); the parity test itself needs no `trf-mod`.
