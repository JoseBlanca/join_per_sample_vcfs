# Hidden-paralog filter — T1 (end-to-end behaviour) + T2 (cost/RSS gate) on tomato2

**Date:** 2026-07-02. **Branch:** `tomato2-paralog-filter`. Milestone T of
[paralog_varcalling_wiring.md](../../implementation_plans/paralog_varcalling_wiring.md),
run after S6c part 2 landed the production two-pass wiring. Validates that the
production `run_var_calling` filter reproduces the isolated-statistics R1 result
([paralog_r1_data_validation_2026-07-01.md](paralog_r1_data_validation_2026-07-01.md))
and measures the cost of the now-default-on path.

## Setup

- **Cohort:** the 59 tomato2 samples, 160 × 200 kb regions (~32 Mb), the same
  fixture R1 used.
- **Reference:** `S_lycopersicum_chromosomes.4.00.fa` (host `$HOME/genomes`,
  bind-mounted read-only into the container via `DEV_EXTRA_MOUNT`).
- **Binary:** `target-container/release/pop_var_caller` (release build).
- **Flags:** `--min-qual 0 --no-allele-balance-filter` (the R1 research-retention
  callset — maximum candidate retention so the paralog class is fully present),
  `--threads 16`. On vs off = with/without `--no-paralog-filter`. Everything else
  identical.
- **Measurement:** wall from `date`; peak RSS from polling `/proc/<pid>/status`
  `VmHWM` (GNU `time` is absent in the dev image); peak spill from polling the
  `.tmp*` files in the output directory.

### A blocker found first: the committed `.psp` carry no summary section

The first T1 run scored **nothing** — `paralog filter — 0 samples fit`,
EM non-converged, fallback π, **0 loci dropped**. The committed
`benchmarks/tomato2/psp_files/*.psp` predate the per-sample summary feature, so
`PspReader::metadata()` returns `None`, `ParalogPrePass::fit` sees only `None`
summaries, and every sample is carried absent (`dump_sample_summary` confirms:
"summary source: re-derived from body"). The filter is then a silent no-op.

T1/T2 below were run on `.psp` **regenerated with the current `pileup`** (which
emits the stored summary section — `dump_sample_summary` now reports "stored
section"), into a scratch directory so the committed fixtures are untouched.

> **Footgun (recommendation).** The filter is on by default, so running it
> against summary-less `.psp` silently produces an *unfiltered* callset (only an
> easily-missed `0 samples fit` + `prior did not converge` on stderr). Recommend
> a loud warning (or hard error) when the filter is requested but a large
> fraction of samples lack a usable coverage summary — "filter INERT; rebuild
> `.psp` with a summary or pass `--no-paralog-filter`". Deferred (not part of
> S6c); noted here for the owner.

## T1 — end-to-end behaviour reproduces R1

| quantity | production (this run) | R1 (isolated stats) |
|---|---|---|
| samples fit | 58 / 59 (1 degenerate carried absent, not fatal) | 59 |
| π (EM) | **0.0992**, `em_converged=true` | ≈ 0.09 |
| Hexp Σ2pq | 27 069 over 376 243 variant loci | — |
| LR cut @ FDR 1 % | 4.05 | — |
| loci dropped | 19 773 / 283 348 = **6.98 %** | — |

**Drop profile** (per-locus, from the on-vs-off VCF diff):

| set | n | mean DP | het fraction | mean AF |
|---|---|---|---|---|
| **dropped** | 19 773 | **492.3 (1.27×)** | **0.124 (2.8×)** | 0.071 |
| kept | 263 575 | 387.6 | 0.045 | 0.065 |

This is the R1 paralog signature: the dropped set carries both **coverage
excess** (1.27× kept, R1 saw 1.31× vs 1.04×) and **heterozygosity excess** (0.124
vs 0.045; R1 saw 0.115 vs 0.043). Allele frequency is *not* the discriminator
(0.071 vs 0.065) — consistent with "the allele-balance filter can't catch the
balanced-het paralog class; coverage is the additive signal." Introgression-like
loci — real variants with normal coverage → negative LR → far from the cut —
sit in the kept set by construction (kept mean DP ≈ 1.0× single-copy).

**T1 verdict: PASS.** The production two-pass wiring reproduces R1's headline
(π ≈ 9 %) and the flagged-set profile (het + coverage excess), with the FDR-1 %
cut dropping ~7 % of the research callset.

## T2 — cost / RSS (the gate)

| config | wall (s) | peak RSS (MB) | peak spill (MB, disk) |
|---|---|---|---|
| `--no-paralog-filter` (baseline) | 23.4 | 336 | 0 |
| filter on (default) | **272.2** | **363** | **3 513** |
| delta | **+248.8 s (+11.6×)** | **+27 (+8 %)** | +3.5 GB disk |

**Memory: FLAT — the gate passes on memory.** Peak RSS rises only +8 % (+27 MB)
despite scoring 376 k loci and spilling everything, because the design holds
nothing per-locus in RAM: the spills are on disk, the LR histogram is a few KB,
the pre-pass + window accumulators are `O(samples)`. RSS does **not** grow with
variant count — the T2 memory obligation holds on real data.

**Wall: +11.6× — a real concern for a default-on path.** The added time is
almost entirely the paralog scoring, not I/O: the 3.5 GB spill read twice (~7 GB,
~15 s) is a small fraction of the +249 s. The per-locus H1/H2 marginal-likelihood
scorer runs over 376 k loci × 58 samples, **twice** (calibrate then write pass),
**single-threaded** — the main caller pass is 16-way parallel, but both scoring
passes run serially after it. That serial double-scan is the bottleneck
(~660 µs / locus / pass).

Mitigating context: this is the *research* callset (`--min-qual 0
--no-allele-balance-filter`), which retains ~5× more candidate loci than a
production run (default `--min-qual 30` + allele-balance on). But the per-locus
cost and the ×2 serial factor remain.

**Spill: 3.5 GB on disk**, dominated by the record spill (the whole
`PosteriorRecord` verbatim, incl. the full genotype-posterior matrix, at
~9 KB/locus × 376 k); the window spill is `O(variant windows × samples)` ≈ tens
of MB. This is the deliberate disk-for-RAM trade (arch §5, ~VCF-scale), and RSS
confirms it stays on disk.

### T2 verdict + recommendations

Memory is flat (the hard obligation holds). The **wall cost is the open
decision** — the plan foresaw exactly this: *"the owner can still choose to ship
it off-by-default if T2 disappoints — a one-line default change."* Options,
cheapest first:

1. **Parallelise the two scoring passes** (rayon over batches of streamed loci) —
   the scorer is pure and per-locus independent; this should recover most of the
   16× the main pass already gets. Highest value, no format change.
2. **Score once, spill the LR** — a third tiny spill (one `f64` per locus,
   ~3 MB) written by the calibrate pass and read by the write pass, instead of
   re-scoring. Halves the scoring cost. It does *not* violate the memory-flatness
   invariant (arch §3 forbids an in-*RAM* genome-wide LR vector; a disk spill is
   fine), but it is a deliberate change to the "recompute, don't cache" rule and
   should be the owner's call.
3. **Ship off-by-default** pending (1)/(2) — the one-line default flip.

Recommendation: pursue (1) (and optionally (2)) as a follow-up before relying on
on-by-default in production; the correctness (T1) is sound, the cost is an
optimisation problem, not a design flaw.

## Reproduction

```
# regenerate summary-bearing .psp (into scratch)
DEV_EXTRA_MOUNT=$HOME/genomes/s_lycopersicum/4.00 ./scripts/dev.sh bash tmp/paralog_t1t2/build_psp.sh
# run on/off with wall+RSS+spill capture
DEV_EXTRA_MOUNT=$HOME/genomes/s_lycopersicum/4.00 ./scripts/dev.sh bash -lc '
  bash tmp/paralog_t1t2/measure2.sh sum_off tmp/paralog_t1t2/sum_off.vcf --no-paralog-filter
  bash tmp/paralog_t1t2/measure2.sh sum_on  tmp/paralog_t1t2/sum_on.vcf'
# drop profile
uv run tmp/paralog_t1t2/profile.py tmp/paralog_t1t2/sum_off.vcf tmp/paralog_t1t2/sum_on.vcf
```
