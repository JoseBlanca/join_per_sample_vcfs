## 6. Findings

### Confirmed correct (no change needed)

- **VCF byte-identity holds.** The worker scores the same `PosteriorRecord` (`Variant = PosteriorRecord`) it then spills, via the same pure `score_spilled_locus` the retired passes used, from bit-identically-built inputs; the spill codec round-trips f64/f32 (NaN survives); the inline histogram is order-independent integer bins → same π/cut → same drops. (reliability, refactor_safety.)
- **`ParalogScoringContext` / `VariantCaller` are genuinely `Sync`** — the context holds only plain read-only data (no `Cell`/`Arc`/atomics); `obs_buf` is per-`call_chunk` local; the histogram is confined to the single sink thread. (unsafe_concurrency.)
- **The merge-join is bit-identical to the old binary search** for hits, misses, duplicate/leading/gap anchors — relying on per-sample position uniqueness (a real invariant). (reliability.)

### Major (both applied)

**M1: worker-scoring path had no test.** Every caller test passed `None` for the context, so the load-bearing M7 fill (`call_chunk` → `paralog_lr`) was never exercised under unit test; a mis-zip/off-by-one would produce a wrong VCF without panicking. **Categories:** reliability. **Fix:** extracted `VariantCaller::score_paralog_lrs(records, window_coverage)` (the fill, minus the `RawCohortChunk` machinery) and added `score_paralog_lrs_fills_one_lr_per_record`, which asserts one LR per record, finite for a scorable locus, `NaN` for an unscored one, each bit-equal to a standalone `ParalogScoringContext::score` (also pinning worker-score == the pure scorer, the reliability Medium-confidence Major).

**M2: stale module docs (convergent).** write_pass.rs:1-18 still called itself "the second spill read … **recomputes** each locus's LR … Recompute-not-cache is the non-negotiable invariant"; calibrate.rs:1-15 still called itself "the first of the two spill reads" whose scorer "the write pass calls to **recompute** each LR bit-for-bit." M7 inverted both. **Categories:** smells (Major ×2), naming (Minor ×2), module_structure (Minor). **Fix:** both headers rewritten to the stored-LR / inline-histogram design; the spill-streaming `calibrate`/`score_spill_record` are now described as `#[cfg(test)]` helpers.

### Minor (all applied)

- **Mi1** pipeline.rs two-pass arm — two `expect`s lacked `// PANIC-FREE:` justification (they're gated by the single `paralog_requested` flag). (errors.) *Fixed.*
- **Mi2** pipeline.rs sink-thread comment said "two-pass = reorder + **accumulate Hexp** + spill" — `Hexp` was retired; the sink now folds the LR histogram. (smells.) *Fixed.*
- **Mi3** `score_spilled_locus` was `pub(crate)` but has only one non-test caller after M7 (all callers now in calibrate.rs). (naming/module_structure cross-note.) *Fixed:* narrowed to a private `fn`.

### Nits (applied)

- pipeline.rs stray apostrophe "its loci' LR" → "its loci's LRs". *Fixed.*

## 7. Out of scope observations

- **`ParalogScoringContext` placement** (module_structure, Medium): it is the worker's type but lives in calibrate.rs to reuse the private `score_spilled_locus`. Kept there (co-location with the shared private scorer is defensible) with the module doc updated to name its dual role; a future `paralog_filter/scoring.rs` split is a reasonable follow-up if the module grows.
- **Per-`call_chunk` `obs_buf` allocation** (reliability/refactor_safety efficiency cross-note): one `Vec` allocated per chunk, reused across the chunk's loci. Coarse-grained (a chunk holds thousands of loci) so amortized; hoisting to per-worker state would fight the `Sync`-by-shared-`&` design. Left as-is; revisit only if a profile flags it.
- **Release-mode monotone desync** (reliability Minor): the merge-join's ascending-input `debug_assert`s are compiled out in release. The preconditions hold by construction (grouper emits strictly-increasing positions; psp rows are coordinate-sorted), so this is defensive, not an active bug; a release-mode check would tax the hot path. Left as `debug_assert`.
- **`cargo doc`** fails on a pre-existing unresolved `ClassicStutterModel` link in the SSR module — unrelated to M7.

## 8. Missing tests added now

- `score_paralog_lrs_fills_one_lr_per_record` (variant_caller.rs) — the M1 fix; see §6.

## 9. What's good

- The `NaN`-as-unscored contract is consistent across all three consumers (worker → `NaN`; `ParalogLrHistogram::push` drops non-finite; `ParalogCalibration::flags` requires finite), so an unscored locus is handled by one uniform path with no special flag. (reliability.)
- The hand-written `CalledChunk::PartialEq` exhaustively destructures (compile-error on a new field) and bit-compares the `NaN`-bearing `paralog_lr`/`window_coverage`, keeping equality correct where a derive would silently break on `NaN`. (refactor_safety.)
- Scoring moved onto the already-parallel workers with the histogram confined to the single sink thread — parallelism with zero added synchronization primitives. (unsafe_concurrency.)
