## 4. Findings & dispositions

- **M1 (reliability) — domain/NaN guard gap. Applied.** A negative/NaN/±inf `theta` slipped the `total <= 0.0` guard: a small negative θ drives `acc[HET]`/`acc[HOM_ALT]` negative while `total > 0`, so `ln(neg/total)` returns a silent `NaN`; the guard comment was inaccurate. Fix (matching the reviewer's recommendation — inputs are once-per-cohort machine-generated hyperparameters, so asserts beat `Result`): `debug_assert` preconditions on `nucleotide_diversity`/`invariant_site_mass` in `new` and on `inbreeding_coefficient` in the method, documented as preconditions; hardened the guard to `!(total > 0.0)` so a non-finite total collapses to hom-ref instead of propagating `NaN`. θ ≥ 0 is guaranteed upstream by `DiversityEstimate` (Step A).
- **M2 (reliability) — missing edge tests. Applied.** Added `zero_diversity_is_certain_homref`, `zero_invariant_mass_still_valid`, `total_zero_collapses_to_homref`, `full_inbreeding_floors_het`, and `folded_sfs_grid_is_symmetric_with_theta_over_p_weights` (the last pins the symmetry the exact 2:1 ratio rests on). 8 → 12 tests.
- **Naming (3 Minor). Applied.** Public params/fields renamed to match Step A's vocabulary: `theta` → `nucleotide_diversity`, `f` → `inbreeding_coefficient`, `invariant_mass`/`DEFAULT_INVARIANT_MASS` → `invariant_site_mass`/`DEFAULT_INVARIANT_SITE_MASS`. Math symbols `p`/`w` kept inside the derivation loop. *Nit `folded_sfs` verb-name — declined* (constructor idiom, à la `Vec::with_capacity`).
- **Visibility (Minor, convergent: smells + idiomatic). Applied.** `FrequencyGrid` and `folded_sfs` made private — internal to `SfsGenotypePrior`, no external consumer.
- **Dead branch (Minor, convergent: smells + idiomatic). Applied.** Removed the `else { 0.5 }` inset fallback via an early return for `n_points == 0`.
- **Defaults (2 Minor). Applied.** Softened the `DEFAULT_SFS_GRID_POINTS` doc (no longer claims an unenforced "matches" equality with the paralog constant); marked `DEFAULT_INVARIANT_SITE_MASS` provisional-pending-calibration.
- **Unused `PartialEq` derive on float structs (Nit). Applied.** Dropped (kept `Debug`, `Clone`).
- **`[f64;3]` + index consts vs a `GenotypeLogPriors` struct (Minor, idiomatic). Deferred.** The `[f64;3]` shape indexed by genotype order is what the posterior engine consumes directly (Step D); a named struct would need converting back at the boundary. Revisit when the engine wiring lands if a struct reads better there.
- **`proptest` for the algebraic laws (Minor, reliability). Deferred.** No `proptest` dependency in the crate; the added deterministic parametrized tests (ratios across F, θ-independence, normalisation) cover the same laws.

## 5. What's good

- The exact `2(1−F)/(1+F)` Wright ratio and its θ-independence are asserted as tight tests, not just approximated — the load-bearing property is pinned.
- Honest reuse of the shared `genetics` primitives (the `exp(log(...))` round-trip is a deliberate anti-drift tradeoff, verified lossless since the grid avoids `p = 0/1`).
- The invariant-mass-vs-ratio decoupling is proven as a test, matching the design claim that the FP/recall knob is separate from the genotype decision.
