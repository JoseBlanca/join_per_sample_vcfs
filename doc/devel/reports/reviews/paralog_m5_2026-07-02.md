# Code review — hidden-paralog filter M5 (per-locus window coverage)

## 1. Scope

- **What:** the M5 milestone diff — switching the paralog score's per-sample
  coverage input from the tile-keyed *window spill* join to per-locus per-sample
  centred-window coverage decoded from the psp `WindowedGc`/`WindowedCoverage`
  columns, gathered in `call_chunk`, carried on the record spill, and read by the
  score.
- **Reviewed against:** `93266f3..HEAD` on branch `tomato2-paralog-filter`
  (commits M5a `debe993`, M5b `4fdc365`, M5c `57f378c`, + rustfmt `539a7b7`).
- **In-scope files:** sample_reader.rs, psp/reader.rs (BlockColumns), types.rs,
  variant_caller.rs, vcf_writer.rs, pipeline.rs, paralog_filter/{spill,
  calibrate, write_pass, test_support}.rs.
- **Out of scope:** the window-spill **write** side (WindowSpillBuilder /
  WindowSpillWriter / feed_window_builder), deliberately left for M6.

## 2. Method

Orchestrated 5 parallel category sub-agents (reliability; refactor_safety;
errors+defaults; naming+idiomatic+smells; module_structure+extras). Verification
(all in the dev container): `cargo build`, `cargo clippy --lib`, `cargo fmt
--check` clean; `cargo test --lib var_calling` = 398 passed; the end-to-end
`var_calling_paralog_filter_two_pass_runs_and_cleans_up` integration test passes.

## 3. Verdict

**Sound, with fixes applied.** No Blockers. The load-bearing invariants all held
under review: the two manual `PartialEq` impls exhaustively destructure and use
`to_bits` (NaN-reflexive) with no `Eq`/`Hash`; the spill encode/decode of the new
`window_coverage` field is an exact mirror; the read-side deletion is complete
and correctly `#[cfg(test)]`-gated; and the calibrate-vs-write-pass bit-identical-LR
invariant is preserved (both now read the same spilled `window_coverage`). The
per-sample direct-indexing sites were audited and proven in-bounds.

## 4. Findings and disposition

### Convergent Major — FIXED

- **M1 `SpillSink::spill_chunk` `resize_with` masked a length mismatch**
  (raised by reliability, refactor_safety, and errors/defaults). `resize_with`
  silently truncates when `window_coverage.len() > records.len()` and pads when
  shorter, so a future gather bug would mis-align per-record coverage with no
  signal. **Fix:** `debug_assert!` the two legal shapes (empty direct-path, or
  `== records.len()`), and pad only the empty case. Also made the `CalledChunk`
  destructure exhaustive (named `_`) to match the diff's own convention.
  `pipeline.rs`.

### Major — FIXED (test gap)

- **M2 grouped/MNP anchor narrowing was untested.** The gather keys on
  `record.locus.start`; a sample covered only at a grouped variant's *interior*
  (no record at the anchor) is left `NaN` → skipped. This is intended (the
  per-position centred window is the design; the direction is safe — fewer
  samples → weaker LR → more likely kept) but was unpinned. **Fix:** added
  `window_coverage_gather_keys_on_anchor_not_group_span` pinning the behaviour.

### Minor — FIXED

- `window_coverage_at` indexed `windowed_gc()[idx]` on an unasserted length
  invariant → now reads via `.get(idx).copied().unwrap_or(NaN)` (a skew degrades
  to "sample absent", never a panic). *(reliability, refactor cross-cat.)*
- Stale/wrong docs (convergent, 4 agents): the `SpillSink` struct doc and
  `spill_chunk` doc still described the retired tile-key join; the
  `gather_window_coverage`/`window_coverage_at` docs named a nonexistent
  `record.locus.pos`. All rewritten to the on-record behaviour / `.start`.
- Duplicate `positions()` / `positions_all()` getters → merged to one
  production `positions()`. *(naming, module.)*
- Bit-NaN f32-slice equality hand-rolled in 4 places → extracted
  `f32_slices_bit_eq` in types.rs, used by both `PartialEq` impls. *(smells.)*
- `spill.rs` module header now records that M5 deliberately re-adds a per-sample
  vector to each record frame (the cost the sibling window spill once avoided),
  with the rationale. *(extras.)*

### Missing tests — ADDED

- `build_observation_skips_on_half_nan_window` — the `gc.is_finite()` half of the
  guard was unexercised (fixtures left gc finite, only NaN-ing coverage).
- `truncated_window_coverage_is_a_typed_error` — pins `decode_f32_vec`'s
  under-read path on the new trailing field.
- `window_coverage_gather_nan_for_legacy_windowless_chunk` — legacy all-NaN
  chunk yields a hit with NaN (skip), never a panic.

### Deferred (documented, not fixed here)

- **Hot-path gather O(loci × samples × log positions) → merge-join
  O(loci + positions)** (extras Major). Both sides are position-sorted, so a
  per-sample forward cursor would drop the `log` factor and re-scan. Deferred to
  **M7**: the gather is per-called-locus (not per-position) and is not the pass's
  dominant cost (the EM is), and M7 moves scoring into the worker where this
  gather is restructured anyway. A per-sample-cursor rewrite (plus a
  monotone-input `debug_assert` restoring the retired `WindowJoin`'s
  loud-on-disorder guarantee) is the intended shape then.
- **Window-spill write side is now write-only / test-only** (WindowSpillRecord,
  WindowSpillWriter, encode_window; the reader is `#[cfg(test)]`). Removed
  wholesale in **M6**.
- `LocusWindowCoverage.gc`/`coverage` vs `windowed_*` elsewhere: kept the short
  names (the type name supplies "window"); added a one-line note that they are
  the same quantities as the psp columns rather than the wider rename.
- `Decoder::*` `.expect(...)` lacking the repo's `// PANIC-FREE:` convention:
  pre-existing, out of scope.

## 5. What's good

- The NaN sentinel is handled consistently everywhere (all compare sites use
  `to_bits`/`is_finite`; no type derives `Eq`/`Hash`).
- The switch keeps the EM/merge core (`MergedRecord`/`PosteriorRecord`)
  untouched, preserving VCF byte-identity; the coverage rides caller-output glue.
- The read-side deletion is clean and the `#[cfg(test)]` gating of the surviving
  reader is a defensible one-milestone bridge with a written M6 removal plan.
