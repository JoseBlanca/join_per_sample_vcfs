## 6. Findings

### Blocker

**B1: src/ssr/mod.rs:13 ‚Äî `cargo doc` build gate broken by unresolved intra-doc links**
**Categories:** reliability, idiomatic, smells, module_structure (+ cross-category from errors, naming, defaults ‚Äî convergent).
**Confidence:** High (reproduced verbatim in ¬ß3).
The design-doc paths are written in `[`backtick-bracket`]` form (`[`doc/devel/specs/ssr_genotyping.md`]`, `[`doc/devel/architecture/`]`), which rustdoc parses as intra-doc links. The crate denies broken intra-doc links, so `cargo doc --no-deps` aborts crate-wide. Both referenced docs exist on disk ‚Äî this is pure markup. Note `types.rs:7` already uses the correct plain-code-span style.
**Fix:** drop the brackets so the paths render as inline code:
```rust
//! Design docs: `doc/devel/specs/ssr_genotyping.md` (the model + why) and
//! `doc/devel/architecture/` (`ssr_genotyping_architecture.md`,
//! `ssr_shared_types.md`, `ssr_catalog.md`).
```

### Major

**M1: src/ssr/types.rs:94-156 ‚Äî `Locus` documented invariants are unenforced; release builds underflow / slice out of bounds**
**Categories:** reliability, errors, refactor_safety, defaults (convergent).
**Confidence:** High.
`Locus` has all-`pub` fields (`types.rs:96-111`) and **no constructor**, yet its doc comment (`types.rs:89-93`) claims the invariant `ref_bytes_start <= start <= end <= ref_bytes_start + ref_bytes.len()` is "upheld at construction." A struct-literal caller (or the Stage-0 builder, once written) can violate it. The only guard is a `debug_assert!` in `tract_offset` (`types.rs:133`) that (a) is compiled out in release and (b) checks only the lower bound. Consequences in release:
- `tract_offset` computes `self.start - self.ref_bytes_start` on plain `u32` ‚Üí **underflow panic** (or wrap, then OOB) when `ref_bytes_start > start`;
- `ref_tract`/`right_flank` slice `ref_bytes` past its end ‚Üí **out-of-bounds slice panic** when `end > ref_bytes_start + ref_bytes.len()`.
Separately, `purity_fraction: f32` is unvalidated (a `NaN` or out-of-`[0,1]` value flows in silently; `is_perfect`'s `== 1.0` then quietly returns `false` for `NaN`).
The input is **external** (TRF output + reference bytes), so "the caller upholds it" is not a safe assumption for a catalog builder.
**Fix:** make the fields private and add a validated constructor returning a typed error, mirroring `Motif::new`:
```rust
pub fn new(chrom: Box<str>, start: u32, end: u32, motif: Motif,
           purity_fraction: f32, ref_bytes: Box<[u8]>, ref_bytes_start: u32)
    -> Result<Self, LocusError>;
```
validating `ref_bytes_start <= start <= end`, `end <= ref_bytes_start + ref_bytes.len() as u32`, and `purity_fraction` finite ‚àà `[0.0, 1.0]`. The accessors then become infallible by construction (the `debug_assert` can stay as documentation). If full encapsulation is too heavy for an internal interchange type (Open Q1), at minimum add a `Locus::validate(&self) -> Result<(), LocusError>` the builder must call, and document the accessors' panic precondition.

### Minor

- **Mi1: src/ssr/types.rs:32 ‚Äî `MotifError` is not `#[non_exhaustive]` and uses a positional field.** (errors) Adding a variant later is a breaking change for any match; a positional `BadLength(usize)` is less self-documenting than `BadLength { len: usize }`. Low impact (internal crate, no external consumers), but cheap to harden now: add `#[non_exhaustive]`.
- **Mi2: src/ssr/types.rs:25-55 ‚Äî `Motif`'s derived `Eq`/`Hash` correctness depends on the zeroed tail.** (refactor_safety, idiomatic, smells) The `#[derive(PartialEq, Eq, Hash)]` compares the whole `buf`; correctness relies on `new` zero-initializing the tail and `0` not being a base. Fields *are* already private and `new` is the only constructor, so the live risk is a *future* constructor or a `pub buf`. Either keep a one-line guard comment on the struct stating "all construction must zero the unused tail ‚Äî derived Eq/Hash depend on it," or remove the coupling entirely by hand-implementing `PartialEq`/`Hash` over `as_bytes()` (then the tail is irrelevant). Lean: the comment ‚Äî the current form is `Copy` and allocation-free, which `smallvec` (not `Copy`) would lose.
- **Mi3: src/ssr/types.rs:141-156 ‚Äî duplicated tract-range arithmetic across `ref_tract`/`left_flank`/`right_flank`.** (smells) Each recomputes `tract_offset()` and the tract length. Extract one `fn tract_range(&self) -> Range<usize>` and slice with it; the three accessors become one-liners and the offset logic lives in one place.
- **Mi4: src/ssr/types.rs:11 ‚Äî `MAX_MOTIF_LEN` documents units but not its source.** (defaults, naming) The doc says "between 1 and this many bases" but not *why 6* (the microsatellite period-‚â§-6 scope). Add the source per the project's no-magic-numbers convention, e.g. `// SSR scope: microsatellite period ‚â§ 6 (spec doc/devel/specs/ssr_genotyping.md ¬ß1.3)`.
- **Mi5: src/ssr/types.rs ‚Äî `pub` vs `pub(crate)` surface.** (defaults) The types are `pub` in an internal crate with no external consumer yet. `pub(crate)` would be more minimal; `pub` is defensible if the CLI/bins will name them. Resolve with Open Q2.
