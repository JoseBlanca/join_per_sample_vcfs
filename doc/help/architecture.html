pop_var_caller — architecture & data flow

Multi-sample SNP caller. The pipeline is run as three discrete steps, each with its own CLI subcommand and on-disk artefact. The authoritative design spec is at doc/devel/specs/calling_pipeline_architecture.md; status board at PROJECT_STATUS.md. Alternate rendering of the same diagram using Mermaid: architecture-mermaid.html.

Three steps at a glance

Solid black arrows show in-process streams; green is shared reference-FASTA access; dashed magenta is the optional Step 2 side-pass and its (c_s, q_b) hand-off into Step 3's posterior engine.

What's in each step

Step 1 — Pileup parallel

One BAM/CRAM in, one .psp out. Streams the BAM exactly once, accumulates per-sample evidence at every covered reference position, applies BAQ in-process, and writes the five freebayes-sufficient scalars per allele (num_obs, q_sum, fwd, placed_left, placed_start) plus phase-chain slot ids.

RolePathNotes
CRAM/BAM input cram_input.rs · cram_files.rs · ref_fetcher.rs Min-MAPQ / min-length / flag filtering; reference fetcher.
pileup walker pileup/ (walker · active_set · cigar_cursor · open_record · slot_allocator) Per-position CIGAR fold with lazy cursor and phase-chain slot allocation. Step-by-step viewer in animations/index.html.

CLI driver pop_var_caller/ — pileup subcommand Shipped.

Step 2 — Contamination estimates optional

Reads the .psp files + a sample-to-batch assignment TSV and estimates per-sample contamination fraction c_s and per-batch contamination-source allele-class distribution q_b via a read-level mixture MLE. Output is a small TOML artefact consumed by Step 3 as frozen inputs (the posterior engine does not re-estimate them). Skipped entirely if --contamination-batches is not supplied; Step 3 then uses c_s = 0 for every sample.

RolePathNotes
batch reader pop_var_caller/batch_assignment.rs Parses the sample → batch TSV.
mixture MLE var_calling/contamination_estimation.rs Read-level EM. Bypasses Step 3's grouping/merger pipeline.
artefact pop_var_caller/contamination_artifact.rs TOML (c_s, q_b) serialiser. Spec: contamination_estimation.md.
CLI driver pop_var_caller/estimate_contamination.rs — estimate-contamination subcommand Shipped.

Step 3 — SNP calling parallel

Joins N .psp readers, filters low-complexity positions, groups overlapping events, reconstructs per-sample likelihoods, runs EM, and writes the multi-sample VCF. Six substages stream end-to-end; the per-group merger and the posterior engine parallelise across independent variant groups via rayon. Only Step 3 needs the reference FASTA (for DUST scoring); only Step 3 consumes the optional (c_s, q_b) from Step 2.

RolePathNotes
per-sample pileup joiner var_calling/per_position_merger.rs Merges the N per-sample .psp streams into a single per-position stream where each record carries every sample's evidence side-by-side.
DUST filter var_calling/dust_filter.rs Streaming sdust filter. No intermediate mask file; reference FASTA consulted on the fly.
variant grouper var_calling/variant_grouping.rs Bundles overlapping / adjacent positions into OverlappingVarGroup values. Sequential, emits independent groups.
per-group merger var_calling/per_group_merger.rs Allele unification + per-sample likelihood reconstruction from the five scalars + phase-chain consistency. Parallel across groups.
Bayesian genotype estimator var_calling/posterior_engine.rs · posterior_engine/ EM over the joined records → per-sample genotype + GQ + QUAL. Mixture E-step when contamination is enabled.
cohort VCF writer var_calling/vcf_writer/ Multi-sample bgzf VCF sink.
CLI driver pop_var_caller/ — cohort subcommand Planned end-to-end wiring.
