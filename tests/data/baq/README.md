# BAQ parity fixtures

Test data for the per-sample-caller BAQ stage. Each `realn0N` triple
exercises `probaln_glocal` against htslib's own production output:

| file | role |
|---|---|
| `realn0N.sam` | input read records (with raw BQ) |
| `realn0N.fa` (+ `.fai` if needed) | reference sequence |
| `realn0N_exp.sam` | expected output: same records plus a `BQ:Z:` tag where each character is `qual[i] - min(qual[i], q[i]) + 64`, computed by `sam_prob_realn(b, ref, len, BAQ_APPLY)` |

Recovering the capped quality from the expected SAM is one subtraction
per base:

```text
bq_baq[i] = qual[i] - (BQ[i] - 64)
```

For non-match positions (soft-clip, insertion, deletion) `BQ[i] = 64`
(zero delta) — htslib only updates BQ at match positions. Our parity
test asserts the recovered `bq_baq` per match position against our own
post-cap output.

## Upstream provenance

All nine files are verbatim copies of:

- `htslib/test/realn01.{sam,fa}`, `htslib/test/realn01_exp.sam`
- `htslib/test/realn02.{sam,fa}`, `htslib/test/realn02_exp.sam`
- `htslib/test/realn03.{sam,fa}`, `htslib/test/realn03_exp.sam`

from htslib commit `2b3ddfd04ec0f8ae57c6f087cf5a916ea4688446` (the
checkout vendored at [`htslib/`](../../../htslib/) in this repo). To
refresh after a future htslib bump, re-copy from the same paths and
update the commit hash here.

## Coverage

The three triples exercise the edges we care about (per the `@CO`
comments in each input SAM):

- `realn01` — reads that overhang the reference; reads whose CIGAR is
  entirely `I` or `S` (no match positions); reads with deletions.
- `realn02` — soft-clipped reads.
- `realn03` — multiple match runs in one CIGAR (e.g. `5=1X5=`) vs. a
  single match run of the same length, used by htslib to assert the
  two are treated equivalently.

### Which fixtures are wired into the parity test

`realn01` and `realn02` are. `realn03` is **not**: its expected SAM is
generated by htslib's `test_realn -e` (extended BAQ;
[htslib/test/test.pl:1496](../../../htslib/test/test.pl#L1496)), which
runs a left/right running-max post-pass on top of `probaln_glocal`'s
output. Extended BAQ is out of scope for this plan (see
[`baq.md`](../../../ia/feature_implementation_plans/baq.md) §"Out of
scope"). The `realn03.*` data files remain here so the corpus is easy
to extend if extended BAQ lands later.

If a category turns out to be under-covered (e.g. a long homopolymer
the htslib fixtures do not include), extend the corpus by running
`samtools calmd -Br` on a synthetic CRAM and committing the input +
expected SAMs alongside the existing files, following the same
naming convention.
