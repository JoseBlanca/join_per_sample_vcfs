## 6. Findings

### Blocker

#### B1: src/cohort/per_group_merger.rs:1118 — `sub_stats` clamps the correct negative q_sum residual to 0
**Categories:** reliability, smells (convergent)
**Confidence:** High

`sub_stats` mutates the constituent's accumulated stats during the scalar-subtraction pass. `q_sum` is documented as `Σ max(ln(P_err_BQ_BAQ), ln(P_err_MQ))` (see [pileup/mod.rs:349](../src/per_sample_caller/pileup/mod.rs#L349)) — a sum of non-positive natural-log error probabilities, so every valid value is `≤ 0`. The code is

```rust
into.q_sum = (into.q_sum - src.q_sum).max(0.0);
```

With `into.q_sum = -50`, `src.q_sum = -15`, the intended residual is `-35`. `f64::max(-35.0, 0.0) = 0.0` — the constituent's q_sum is silently zeroed. Downstream, `standard_log_likelihood` adds `stats.q_sum` to `log_l` for alleles `a ∉ G`; with q_sum forced to 0 every chain-anchored sample gets a wrong likelihood for any genotype that excludes the constituent. No test reads the post-subtraction value, which is why the bug survived. **Fix:** swap to `.min(0.0)` (the actual intent is "do not let the residual go *positive*"). Add a regression test on `sub_stats` directly.

#### B2: src/cohort/per_group_merger.rs:845 — `enforce_max_alleles` emits a 1-allele record when `max_alleles ≤ 1`
**Categories:** reliability
**Confidence:** High

`process_group`'s early-exit `if unified.alleles.len() < 2 { return Ok(None); }` is bypassed by the appended ghost. With `max_alleles = 1`, REF stays protected, every alt folds into the ghost; the set becomes `[REF, ghost]`, length 2; the `< 2` check passes; the post-cap filter drops the ghost; the emitted record carries `alleles == [REF]`. Same path for `max_alleles = 0`. **Fix:** either validate `max_alleles >= 2` at construction (or in `process_group`), or re-check the *real* allele count after the cap:

```rust
let real_alleles = unified.alleles.iter()
    .filter(|a| !a.seq.is_empty() || a.is_compound)
    .count();
if real_alleles < 2 { return Ok(None); }
```

Add `cap_one_returns_none_no_record_emitted`.

### Major

#### M1: src/cohort/per_group_merger.rs:889 — Ghost sentinel `UnifiedAllele` smuggles the OTHER pool through the type
**Categories:** smells, idiomatic, refactor_safety (cross), naming (cross)
**Confidence:** High

The OTHER pool is encoded as a synthetic `UnifiedAllele` with empty `seq` appended to `alleles`. Four downstream sites re-derive the same predicate (`project_scalars`, `build_ca_flags`, `compute_log_likelihoods`, plus `process_group`'s alleles filter). The author comment at line 887 admits the workaround. **Fix:** lift the OTHER bucket to a peer field on `UnifiedAlleleSet`:

```rust
struct UnifiedAlleleSet {
    alleles: Vec<UnifiedAllele>,
    other_per_sample_sources: Vec<Vec<(usize, usize)>>,
}
```

`enforce_max_alleles` populates it directly, the three downstream helpers drop their `has_ghost` predicate, the final filter at `:439` disappears.

#### M2: src/cohort/per_group_merger.rs:1088 — `find_per_position_allele_idx` is O(n_alleles × n_sources) inside a triple loop
**Categories:** smells
**Confidence:** High

The subtract-compound-from-constituents pass calls `find_per_position_allele_idx` on every constituent-source iteration, and the function itself walks every allele's `per_sample_sources`. **Fix:** build a `(sample_idx, record_idx, local_allele_idx) → kept_idx` `HashMap` once per `project_scalars` and look up in O(1).

#### M3: src/cohort/per_group_merger.rs:1108 + 272 / per_position_merger.rs:137 / variant_grouping.rs:120 — Field enumeration without exhaustive destructure
**Categories:** refactor_safety
**Confidence:** High

`add_stats`, `sub_stats`, and the three manual `Debug` impls hand-pick fields by name. A new field on `AlleleSupportStats` or on any of the three merger structs silently fails to flow through. **Fix:** force-exhaustive destructure (preferred: add `AddAssign`/`SubAssign` on `AlleleSupportStats` with the destructure done once), use `let Self { … } = self;` in every manual `Debug` impl, replace `..Default::default()` in the two test configs (`:1847`, `:1960`) with explicit literals.

#### M4: src/cohort/per_group_merger.rs:1012 + 803 + 1051 + 1287 — Silent contract-violation fallbacks
**Categories:** errors
**Confidence:** High (lines 1012, 803), Medium (1287, 1051)

Each site silently substitutes a fallback value when an internal invariant fails:
- `:1012` `min_mean_q.unwrap_or(0.0)`: chain-anchored compound with `inter > 0` but every constituent has `num_obs = 0` ⇒ `q_sum = 0`, infinite-quality compound smuggled into the likelihood.
- `:803` `unwrap_or_default()`: compound projection silently produces empty bytes if no sample has the requested `(record_idx, local_allele_idx)`.
- `:1051` `support.num_obs == 0 ⇒ continue`: walker invariant says non-REF buckets have `num_obs ≥ 1`; the silent skip would hide a real upstream contradiction.
- `:1287` `p_counts.get(&a).unwrap_or(&0)`: contract holds today; if it ever breaks, `xlogy` returns `-inf` and bypasses the NaN/+inf degeneracy check.

**Fix:** either `tracing::warn!`/typed-error the inconsistency, or annotate `// PANIC-FREE:` naming the upstream invariant proving the fallback is unreachable.

#### M5: src/cohort/per_group_merger.rs:434, 708, 1379 — Three dead `let _ = …;` orphans
**Categories:** idiomatic, smells (convergent), errors (cross), naming (cross)
**Confidence:** High

- `let _ = n_samples;` at 434: `n_samples` is `Copy`, was used twice, no warning to suppress.
- `let _ = group_start;` at 708: `group_start = group.start` at 665 is otherwise never used; the entire pair is dead.
- `let _ = &mut counts;` at 1379: `counts` declared `mut` but never mutated; the dead reborrow exists to suppress a non-existent `unused_mut` warning.

**Fix:** delete the three lines (and the matching `mut` on `counts`, and the `let group_start = …;` at 665). One of the three may have been the abandoned start of a missing compound-subtraction step inside `chain_broken_log_likelihood` — verify before deleting.

#### M6: src/cohort/per_group_merger.rs:493 — `UnifiedAlleleSet::ref_idx` is `#[allow(dead_code)]` and hard-coded to 0
**Categories:** idiomatic, smells (convergent), naming (cross)
**Confidence:** High

Field is never read, doc says "always 0", every callsite uses `alleles[0]` directly. **Fix:** delete the field and the `#[allow]`. Document the REF-at-0 invariant in the struct's doc comment.

#### M7: src/cohort/per_group_merger.rs:289 + variant_grouping.rs:145 — Zero-config `new()` constructors hide behavioral defaults
**Categories:** defaults
**Confidence:** High

`PerGroupMerger::new` defaults `ploidy = 2`, `max_alleles = 6`, `batch_size = 32`; `VariantGrouper::new` defaults `max_var_group_span = 10_000`. None of these are visible at the call site. **Fix:** either delete the `new` constructors so every caller writes `with_config(.., Default::default())`, or rename to `with_default_config`.

#### M8: src/cohort/per_group_merger.rs:394 — `process_group` underflows `u32` if `end < start`
**Categories:** reliability
**Confidence:** High

`let span = end - start + 1;`. `OverlappingVarGroup` is a `pub` struct with no construction invariant; future producers other than `VariantGrouper` could feed inverted ranges. **Fix:** validate at entry and emit a typed error, or change `OverlappingVarGroup` to enforce `start <= end` via a constructor.

#### M9: src/cohort/per_group_merger.rs:957, 1042, 1321 — Direct `pp.per_sample[sample_idx]` panics on heterogeneous widths
**Categories:** reliability
**Confidence:** Medium

`n_samples` is derived from `group.records.first().per_sample.len()` only. If any other record's `per_sample` is shorter, indexing panics inside a rayon worker. **Fix:** debug-assert on entry that every `pp.per_sample.len() == n_samples`, or use `.get(sample_idx).and_then(|s| s.as_ref())` at the three sites.

#### M10: src/cohort/per_group_merger.rs:657 — Compound candidates byte-keyed via zero pseudo-ref can collide
**Categories:** reliability
**Confidence:** Medium

The byte-key uses `project_compound_onto_group(&pseudo_ref, …)` where `pseudo_ref = vec![0u8; ref_span_total]`. If two distinct local-allele indices produce identical bytes in the same record (pathological but constructible), they map to the same candidate. **Fix:** key candidates by the canonical `Vec<(record_idx, local_allele_idx)>` tuple sorted; that 1-1 maps to byte sequences under the walker invariant.

#### M11: src/cohort/per_group_merger.rs:396 — `RefSeqFetcher::fetch` return length not validated
**Categories:** reliability
**Confidence:** High

The trait doc says exactly `length` bytes; the merger never checks. A short fetch leads to a panic in `project_local_allele`'s slice. **Fix:** debug-assert or explicit error after the fetch.

#### M12: src/cohort/per_group_merger.rs:213, 1376 — `ploidy = 0` is unhandled
**Categories:** reliability
**Confidence:** Medium

No validation in `PerGroupMergerConfig` rejects `ploidy = 0`. `genotype_order(0, n)` returns `[[]]`; `chain_broken_log_likelihood` divides by `n_slots = 0`. **Fix:** reject at construction or assert in `process_group`.

#### M13: src/cohort/per_group_merger.rs:640 — `CompoundSampleInfo` uses the blacklisted `Info` generic noun
**Categories:** naming
**Confidence:** High

**Fix:** rename to `CompoundChainAnchorEvidence` (or `PerSampleChainAnchor`) and update the binding `sample_info` to match.

#### M14: src/cohort/per_position_merger.rs:54, 76 — `MergerError` / `Reader` are mechanism-named and inconsistent with sibling `PerGroupMergerError`
